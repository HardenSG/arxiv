## 一、研究背景与动机

### 1.1 现有工作的不足

当前 Agentic AI 在实际应用中频繁失败，主要源于大型语言模型（LLM）的五大固有局限：

1. **上下文约束（Context Constraints）**：LLM 的上下文窗口有限，无法处理超长代码库或跨会话的复杂任务
2. **跨会话遗忘（Cross-Session Forgetting）**：多轮对话中，早期获得的关键信息容易被后续token稀释
3. **随机性（Stochasticity）**：LLM 输出的不确定性导致相同提示可能产生不同结果
4. **指令执行失败（Instruction Failure）**：模型对复杂指令的理解和遵循能力有限
5. **适应刚性（Adaptation Rigidity）**：模型难以快速适应新领域或新任务

### 1.2 研究空白

现有研究主要集中在通过**提升模型能力**来解决上述问题，如扩展上下文窗口、增加参数规模等。然而，作者指出这些方法是**必要但不充分**的——模型容量的提升无法从根本上解决结构性问题。

### 1.3 与已有工作的关系

本文提出的**双螺旋治理框架（Dual-Helix Governance Framework）**将上述五大局限重新定义为**结构性的治理问题**，认为需要超越模型本身，从系统架构层面引入外部化机制来保障 Agentic AI 的可靠性。

### 3.2 知识轨（Knowledge Track）

**核心思想**：将领域知识从 LLM 内部转移到外部知识图谱

- **知识图谱底层**：使用图数据库存储结构化的领域事实
- **可执行协议（Executable Protocols）**：将业务规则编码为可验证的逻辑规则
- **优势**：
  - 解决上下文约束：知识不依赖 prompt 长度
  - 解决跨会话遗忘：知识持久化存储
  - 解决适应刚性：新知识可直接注入图谱

### 3.3 行为轨（Behavior Track）

**核心思想**：对 Agent 的行为进行实时监控和约束

- **执行协议 enforcement**：定义行为边界和约束条件
- **实时监控**：跟踪 Agent 的决策链路
- **干预机制**：在越界行为发生时进行自动干预
- **优势**：
  - 解决随机性：通过确定性规则约束输出空间
  - 解决指令失败：协议提供明确的执行路径

### 3.4 技能轨（Skills Track）

**核心思想**：标准化 Agent 的能力单元

- **技能库管理**：定义和管理可复用的技能单元
- **持续学习循环**：支持从实践中自动获取新技能
- **优势**：
  - 提升任务泛化能力
  - 支持渐进式能力增长

### 3.5 自学习循环机制

框架包含一个**闭环的自学习系统**：

1. **经验收集**：从实际任务执行中收集案例
2. **知识提炼**：从成功案例中提取可复用的知识
3. **知识注入**：将提炼的知识更新到知识图谱
4. **技能进化**：基于新知识更新技能库
